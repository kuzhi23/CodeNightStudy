[프론트엔드] 1. 쿠키와 웹스토리지의 차이점을 설명해라

```
일반적으로 http 특징은 무상태, 비연결성이라고 할 수 있습니다. 이 것은 서버와 클라이언트 사이에 이루어지는 요청/응답이 일회성이고 데이터가 저장안된다는 것을 말해주고 있습니다. 이점을 보완하고자 나온게 위 3개라고 할 수 있습니다.

먼저 쿠키를 살펴보면 쿠키는 서버가 사용자의 웹브라우저에 전송하는 만료기간이 있는 작은 데이터 조각들입니다. 위의 두개 처럼 키와 밸류값을 갖는다는 특징이 있고, 서버는 이 쿠키들로 세션관리, 개인화, 트랙킹을 할 수 있습니다. 쿠키의 종류에도 session cookie와 persistant cookie가 있는데 session cookie는 윈도우 탭이나 창을 닫으면 없어지지만 persistant cookie는 따로 만료일자를 지정해 탭이나 창을 닫아도 사라지지않습니다.

다음 Webstorage인 localStorage와 sessionStoage를 간단히 설명하자면 이 두개는 HTML5에서 추가된 저장소 이며 쿠키처럼 간단한 키와 값을 저장할 수 있는 키-밸류형태의 스토리지 입니다.

쿠키와의 WebStorage의  가장 큰 차이점은 매번 서버에 데이터를 보내지 않는다는 것과 용량의 차이입니다.  localStorage를 사용하면 http요청에서 데이터를 주고 받을 수는 없지만 클라이언트와 서버간의 전체 트래픽과 낭비되는 대역폭의 양을 줄일 수 있고, 인터넷이 끊어져도 데이터가 삭제되지않습니다. 또한 쿠키가 보유할 수 잇는 4kb의 데이터보다 최대 5mb의 정보를 저장할 수 있습니다.

위 두개의 차이점은 localstorage는 저장한 데이터를 명시적으로 지우지 않는 이상 영구적으로 보관이 가능하지만 session storage는 session cookie처럼 탭이나 윈도우를 닫으면 없어진다는 특징이 있습니다.
```

## 요약

쿠키는 더 작고 모든 http요청과 함께 서버 정보를 다시 전달해주지만 webstorage는 더 크고 클라이언트 측에 정보를 보유할 수 있다.
